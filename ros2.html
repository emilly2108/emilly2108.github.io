ROS 2란 무엇인가? 기본 개념 정리

2026.05.24

안녕하세요! 오늘은 로봇 공학의 핵심 미들웨어인 ROS 2(Robot Operating System 2)의 기본 개념과 핵심 아키텍처에 대해 정리해 보려고 합니다.

로봇을 개발할 때 모터 제어, 센서 데이터 수집, 인지 및 주행 알고리즘을 처음부터 끝까지 혼자 만드는 것은 거의 불가능에 가깝습니다. ROS 2는 이러한 복잡한 기능을 서로 쉽게 연결해 주는 로봇용 미들웨어 시스템입니다.

"ROS는 운영체제(OS)가 아니라, 로봇 소프트웨어 개발을 위한 오픈소스 '미들웨어' 프레임워크입니다."

1. ROS 1 vs ROS 2의 가장 큰 차이점

기존 ROS 1은 단일 마스터 시스템 기반이었기 때문에 마스터가 죽으면 로봇 전체 통신이 마비되는 치명적인 약점이 있었습니다. 하지만 ROS 2는 이를 보완하기 위해 DDS(Data Distribution Service)를 표준 통신 기술로 도입했습니다.

핵심 변경 사항 요약:

  • 마스터 없는 구조: 데이터 분산 서비스(DDS) 덕분에 중앙 서버 없이 로봇 내 노드들이 서로를 자동으로 탐색합니다.
  • 실시간성 보장 (Real-time): 저수준 제어나 정밀한 하드웨어 동기화가 가능해졌습니다.
  • 멀티 로봇 지원: 여러 대의 로봇이 무선 네트워크 환경에서도 끊김 없이 데이터를 주고받을 수 있습니다.

2. ROS 2의 기본 통신 방식 종류

ROS 2 시스템은 수많은 '노드(Node)'들의 집합으로 이루어집니다. 이 노드들이 소통하는 방식은 크게 세 가지로 나뉩니다.

① 토픽 (Topic)

연속적인 데이터를 비동기식으로 주고받을 때 사용합니다. 카메라 영상 스트리밍이나 라이다(LiDAR) 센서 데이터처럼 끊임없이 발행(Publish)하고 구독(Subscribe)하는 형태입니다.

② 서비스 (Service)

요청(Request)과 응답(Response) 구조를 가진 일회성 동기식 통신입니다. "로봇 현재 위치 알려줘!"라고 요청하면 "지금 X: 1.2, Y: 0.5 야"라고 대답해 주는 방식입니다.

3. 간단한 명령어 예시

터미널에서 현재 활성화된 로봇 노드들과 토픽 목록을 확인하는 가장 기초적인 명령어입니다.

# 활성화된 노드 목록 확인하기
$ ros2 node list

# 현재 발행 중인 토픽 목록 확인하기
$ ros2 topic list

앞으로 이 블로그를 통해 다양한 로봇 시뮬레이션 환경(Isaac Sim, Gazebo 등)과 커스텀 하드웨어를 연동하는 과정을 차근차근 기록해 나갈 예정이니 기대해 주세요!
