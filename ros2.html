ROS 2란 무엇인가? 핵심 개념과 통신 방식 정리

2026-05-24

ROS 2의 역할과 ROS 1과의 차이, 노드·토픽·서비스를 중심으로 기본 통신 구조를 정리합니다.

로봇을 개발할 때는 모터 제어, 센서 데이터 수집, 인식, 경로 계획처럼 서로 다른 기능을 동시에 다뤄야 합니다. 이 모든 기능을 처음부터 하나의 프로그램으로 만드는 일은 복잡하고 유지보수도 어렵습니다.

ROS 2(Robot Operating System 2)는 이런 기능을 독립적인 프로그램으로 나누고 서로 통신하게 만드는 로봇 소프트웨어 미들웨어입니다. 이름에 Operating System이 들어가지만 Windows나 Linux 같은 운영체제 자체는 아닙니다.

ROS 2는 로봇 기능을 작은 구성 요소로 분리하고, 각 요소가 표준화된 방식으로 데이터를 주고받도록 돕는 개발 프레임워크입니다.

ROS 1과 ROS 2의 핵심 차이

ROS 1은 중앙의 ROS Master를 통해 노드가 서로를 찾는 구조를 사용합니다. 구조가 단순하다는 장점이 있지만, 중앙 구성 요소에 문제가 생기면 시스템 전체의 통신에 영향을 줄 수 있습니다.

ROS 2는 DDS(Data Distribution Service) 기반 통신을 도입했습니다. 각 노드가 네트워크에서 서로를 발견할 수 있어 중앙 Master 없이도 분산된 시스템을 구성할 수 있습니다.

  • 분산 구조: 중앙 Master에 의존하지 않고 노드가 서로를 발견합니다.
  • QoS 설정: 데이터의 신뢰성, 보관 방식, 전송 지연 요구 등을 상황에 맞게 조정할 수 있습니다.
  • 다양한 플랫폼: Linux뿐 아니라 Windows와 macOS 등도 고려합니다.
  • 실시간 시스템 지원: 예측 가능한 응답 시간이 중요한 로봇 제어 환경을 더 잘 지원하도록 설계됐습니다.

ROS 2 시스템의 기본 단위: 노드

ROS 2 애플리케이션은 여러 개의 노드(Node)로 구성됩니다. 노드는 특정 역할을 담당하는 하나의 실행 단위입니다. 예를 들어 카메라 노드는 영상을 발행하고, 객체 인식 노드는 그 영상을 받아 처리할 수 있습니다.

기능을 노드 단위로 분리하면 각 부분을 독립적으로 개발하고 테스트하기 쉬워집니다. 한 노드를 다른 구현으로 교체하거나, 여러 컴퓨터에 나누어 실행하는 것도 가능합니다.

대표적인 통신 방식

토픽(Topic)

토픽은 연속적인 데이터를 비동기 방식으로 전달할 때 사용합니다. 데이터를 보내는 노드를 Publisher, 받는 노드를 Subscriber라고 합니다. 카메라 영상, LiDAR 스캔, 로봇 속도 명령처럼 계속 흐르는 데이터에 적합합니다.

서비스(Service)

서비스는 요청(Request)과 응답(Response)이 한 쌍인 통신 방식입니다. “현재 로봇의 상태를 알려줘”처럼 특정 시점에 결과 하나를 받고 싶을 때 사용합니다. 요청을 보낸 쪽은 응답이 돌아올 때까지 기다리는 형태로 사용할 수 있으므로, 지속적인 데이터 스트림에는 토픽이 더 적합합니다.

액션(Action)

액션은 시간이 오래 걸리는 작업을 시작하고, 진행 상황을 받으며, 필요하면 취소할 수 있는 통신 방식입니다. 로봇을 특정 위치로 이동시키는 작업처럼 완료까지 시간이 필요한 명령에 잘 맞습니다.

기본 명령어로 구조 확인하기

ROS 2 환경이 준비됐다면 다음 명령으로 현재 실행 중인 노드와 토픽을 확인할 수 있습니다.

# 실행 중인 노드 목록
ros2 node list

# 현재 사용 중인 토픽 목록
ros2 topic list

# 특정 토픽의 데이터 확인
ros2 topic echo /topic_name

정리

ROS 2는 복잡한 로봇 시스템을 여러 노드로 나누고, 토픽·서비스·액션으로 연결하게 해 줍니다. DDS 기반의 분산 구조와 QoS 설정은 여러 장치가 협력하는 실제 로봇 환경에서 특히 중요합니다.

처음에는 노드와 토픽의 관계부터 익힌 뒤, 간단한 Publisher와 Subscriber를 직접 만들어 보는 것이 좋습니다. 그다음 서비스와 액션, QoS 설정으로 범위를 넓히면 ROS 2의 통신 구조를 한층 자연스럽게 이해할 수 있습니다.
