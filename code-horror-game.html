유니티를 활용한 공포게임 협업 작품

2025.07–2025.08

2025년에 약 1개월간 친구와 함께 제작한 첫 협업 작품입니다. Raycast 기반 상호작용, 상태값 기반 퍼즐 흐름, ScriptableObject 아이템 데이터, Coroutine/Lerp 애니메이션으로 플레이어가 퍼즐과 사물 변화를 자연스럽게 체감하도록 구현했습니다.

이 프로젝트는 Unity를 활용해 만든 3D 공포게임 협업 작품입니다. GitHub 기록 기준 저장소는 2025년 7월 14일에 생성되었고, 마지막 주요 작업은 2025년 8월 18일에 반영되어 약 한 달 동안 집중적으로 제작했습니다.

  • 작업 기간 2025년 7월 14일 ~ 2025년 8월 18일
  • 협업 방식 친구와 함께 역할을 나누어 제작한 Unity 공포게임 프로젝트
  • 주요 기술 Unity, C#, Raycast, TextMeshPro UI, Coroutine, ScriptableObject, Prefab/Scene workflow

구현 방식

  • Camera.main.ScreenPointToRay와 Physics.Raycast를 사용해 화면 중앙 기준으로 상호작용 가능한 문, 서랍, 퍼즐, 아이템을 감지했습니다.
  • 오브젝트 종류는 Tag와 SerializedField 참조로 구분하고, 플레이어가 Z/R 키를 누를 때 UI 표시, 이동 잠금, 퍼즐 창 열림/닫힘이 함께 동작하도록 연결했습니다.
  • Showing, Solve, opening 같은 상태값을 분리해 퍼즐 진행 단계와 오브젝트 열림 상태를 간단한 FSM처럼 관리했습니다.
  • 아이템은 ScriptableObject 기반 Item 데이터로 이름, 설명, 타입, 이미지, 프리팹을 분리하고, Inventory/Slot 구조로 획득 흐름을 구현했습니다.
  • 문과 서랍 애니메이션은 Coroutine, Quaternion.Lerp, Vector3.Lerp를 사용해 시간 기반으로 부드럽게 열리고 닫히도록 만들었습니다.

기술 스택을 실제 기능에 연결한 부분

핵심 구현은 C# 게임 로직입니다. TextMeshPro는 비밀번호·안내 문구 UI에 사용했고, ShaderLab/HLSL 파일은 주로 TextMeshPro와 에셋 렌더링 리소스로 포함되어 있어 담당 기능 설명에서는 “직접 작성한 셰이더”처럼 과장하지 않고 구분했습니다.

협업과 버전 관리

친구와 역할을 나누어 작업하면서 GitHub로 씬, 프리팹, 스크립트를 공유했습니다. Unity 프로젝트는 씬과 프리팹 충돌이 생기기 쉬워서, 내가 맡은 퍼즐·상호작용·애니메이션 스크립트 단위로 기능을 나누고 SerializedField로 필요한 오브젝트를 연결하는 방식으로 작업 범위를 분리했습니다.

첫 협업 작품이라 완성도뿐 아니라 “어떤 기능을 누가 책임지고, 어떻게 씬에 연결하며, 변경 사항을 어떻게 합칠지”를 경험할 수 있었던 프로젝트입니다.
